Backend Flask + Frontend Flet + Tailwind CSS

Um catálogo de filmes com API documentada, interface desktop e landing page.

Este projeto entrega uma aplicação full stack em Python para listar filmes, consultar detalhes por ID e cadastrar novos títulos com validação Pydantic. A documentação Swagger, o frontend em Flet e esta página em Tailwind completam a experiência.

Visão geral

Três partes integradas em uma única entrega

Backend

Flask com Blueprint

A API foi organizada com Blueprint, expõe endpoints REST para listagem, consulta por ID e criação de filmes, e usa Flask-RESTX para documentação Swagger.

Frontend

Flet consumindo a API

A aplicação desktop lista os dados vindos do endpoint GET, permite cadastrar novos filmes pelo POST e apresenta feedback visual ao usuário a cada ação.

Landing page

Tailwind CSS para apresentação

Esta página resume o objetivo do projeto, mostra os requisitos atendidos e explica exatamente como executar cada parte da aplicação.

Checklist

Requisitos da atividade cobertos

Backend

  • Blueprint configurado em backend/api/routes.py
  • Dois endpoints GET documentados com Swagger e docstrings
  • Endpoint POST validando payload com Pydantic
  • Swagger UI disponível em /api/docs

Frontend + Landing

  • Tela Flet consumindo o GET para listar os filmes
  • Formulário Flet enviando dados para o POST com feedback ao usuário
  • Landing page estática em HTML com Tailwind CSS
  • Nome, descrição e instruções de execução apresentados nesta página

API

Endpoints principais

GET /api/movies

Lista todos os filmes cadastrados no catálogo.

GET /api/movies/<id>

Retorna os dados de um filme específico pelo identificador. Se não existir, responde com 404.

POST /api/movies

Cadastra um novo filme com validação de título, gênero, ano e nota usando Pydantic.

Exemplo de payload

{
  "title": "Matrix",
  "genre": "Ficção científica",
  "year": 1999,
  "rating": 8.7
}

Exemplo de resposta com erro

{
  "error": [
    {
      "loc": ["title"],
      "msg": "String should have at least 1 character"
    }
  ]
}

Execução

Como rodar o projeto

1

Instale as dependências

python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
2

Inicie o backend

python -m backend.app

API em http://127.0.0.1:5000/api
Swagger em http://127.0.0.1:5000/api/docs

3

Abra o frontend Flet

flet run frontend/main.py

A interface consome o GET para listar os filmes e o POST para cadastrar novos títulos.
