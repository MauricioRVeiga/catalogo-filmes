API REST Documentada

Flask-RESTX gera automaticamente uma interface Swagger UI completa para explorar e testar todos os endpoints.

  • → Respostas tipadas e documentadas

Interface Desktop

Frontend multiplataforma construído com Flet em Python puro. Sem HTML, sem JS — UI rica direto do Python.

  • → Windows, macOS e Linux
  • → Conectado ao backend Flask
  • → 100% Python

  • → Tipos e limites verificados
  • → Erros 400 com detalhes do campo
  • → Schemas reutilizáveis

Como executar

Três passos para ter o projeto rodando localmente.

1

Dependências

Crie um ambiente virtual e instale os pacotes necessários.

# criar venv

python -m venv .venv


# ativar (Windows)

.venv\Scripts\activate


# instalar pacotes

pip install -r requirements.txt

2

Backend

Inicie o servidor Flask. A API e o Swagger ficam disponíveis.

cd backend

python app.py


# API disponível em:

http://127.0.0.1:5000/api


# Swagger em:

http://127.0.0.1:5000/api/docs

3

Frontend

Abra a interface desktop Flet em um novo terminal.

# em outro terminal

cd frontend

flet run main.py


# janela desktop abre

# conectada ao Flask

Endpoints da API

Documentação interativa em /api/docs (Swagger UI).

GET

/api/movies

200 OK

Lista todos os filmes cadastrados. Retorna um array JSON com todos os registros do catálogo.

GET

/api/movies/{id}

200 · 404

Retorna os detalhes de um filme pelo ID. Devolve 404 com mensagem de erro se o ID não existir.

POST

/api/movies

201 · 400

Cadastra um novo filme. Todos os campos são validados com Pydantic antes de persistir.

title

string · 1–100

genre

string · 1–60

year

int · 1888–2100

rating

float · 0.0–10.0

400

Exemplo de resposta quando a validação Pydantic falha — title enviado vazio

Request body

{
  "title":  "",
  "genre":  "Ação",
  "year":   1999,
  "rating": 8.7
}

400 Bad Request · application/json

{
  "error": [{
    "type":  "string_too_short",
    "loc":   ["title"],
    "msg":   "at least 1 character",
    "input": ""
  }]
}
